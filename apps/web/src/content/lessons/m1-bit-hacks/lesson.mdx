# Bit Hacks: Reverse, Parity, CLZ, and Powers of Two

Beyond set/clear/toggle, a handful of **bit hacks** show up constantly in systems code, interviews,
and CS:APP's DataLab: reversing bits, computing parity, counting leading zeros, and rounding to a
power of two. Each is a small puzzle with a slick solution — and each maps to a real hardware
instruction or systems use.

## Reversing bits

Reversing the bit order of a word (bit 0 ↔ bit 31, etc.) appears in DSP (FFT bit-reversal
permutation), cryptography, and some hardware protocols. The straightforward method walks each bit:

```c
uint32_t reverse_bits(uint32_t x) {
  uint32_t r = 0;
  for (int i = 0; i < 32; i++) { r = (r << 1) | (x & 1u); x >>= 1; }
  return r;
}
```

A faster approach swaps bit *groups* (nibbles, bytes, halves) in log steps — the classic
"divide-and-conquer" bit reversal — but the loop version is perfectly clear for learning.

## Parity

**Parity** is the XOR of all bits: 1 if the number of set bits is odd, else 0. It's the basis of
simple error detection (parity bits) and relates to popcount mod 2.

```c
int parity(uint32_t x) {
  int p = 0;
  while (x) { p ^= 1; x &= (x - 1); }  // clear lowest set bit each step
  return p;
}
```

A branch-free fold (`x ^= x >> 16; x ^= x >> 8; ...`) computes it in log steps — the same
divide-and-conquer idea.

## Counting leading zeros (CLZ)

**CLZ** counts the zero bits before the highest set bit — e.g., `clz(1) = 31`, `clz(0x80000000) =
0`. It's how you compute `floor(log2(x))` (= `31 - clz(x)`), size buckets in allocators, and
normalize floats. Most CPUs have a dedicated `lzcnt`/`clz` instruction; `__builtin_clz` maps to it.

```c
int count_leading_zeros(uint32_t x) {
  if (x == 0) return 32;
  int n = 0;
  while ((x & 0x80000000u) == 0) { n++; x <<= 1; }
  return n;
}
```

## Rounding up to a power of two

Allocators and hash tables love power-of-two sizes (so they can mask instead of mod — recall the
ring buffer). To round `x` **up** to the next power of two, the "smear the bits down" trick fills all
bits below the top set bit, then adds one:

```c
uint32_t next_pow2(uint32_t x) {
  if (x <= 1) return 1;
  x--;                      // handle exact powers of two
  x |= x >> 1;  x |= x >> 2;  x |= x >> 4;
  x |= x >> 8;  x |= x >> 16;   // now x is all-ones up to the top bit
  return x + 1;
}
```

Alternatively, `1u << (32 - clz(x - 1))` using CLZ. Either way you get the smallest power of two
≥ x.

## Gray code

**Gray code** orders numbers so that consecutive values differ by exactly **one bit** — used in
rotary encoders, error correction, and Karnaugh maps to avoid glitches during transitions. Binary
→ Gray is a one-liner:

```c
uint32_t binary_to_gray(uint32_t x) { return x ^ (x >> 1); }
```

For example, 0,1,2,3 → 0,1,3,2 (each step flips one bit). The inverse (Gray → binary) XORs down the
shifted values.

## Recap

- Reverse bits (bit i ↔ 31-i); parity = XOR of all bits (odd/even set-bit count).
- CLZ counts leading zeros; `floor(log2(x)) = 31 - clz(x)`.
- Round up to a power of two by smearing bits down then `+1` (or via CLZ).
- Gray code (`x ^ (x>>1)`) makes consecutive values differ by one bit.
