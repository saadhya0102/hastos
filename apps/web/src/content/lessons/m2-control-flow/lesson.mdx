# Control Flow and Condition Codes

High-level `if`, `while`, `for`, and `switch` all compile down to two primitives: **setting
condition flags** and **jumping** based on them. Understanding this layer demystifies branching,
explains signed-vs-unsigned comparison bugs at the machine level, and reveals branchless tricks.

## Condition flags

The CPU keeps a set of single-bit **condition codes** (flags) describing the result of the most
recent arithmetic/logical operation. The important ones:

- **ZF** (zero) — the result was zero.
- **SF** (sign) — the result's most significant bit was 1 (negative, for signed).
- **CF** (carry) — an unsigned carry/borrow occurred (unsigned overflow).
- **OF** (overflow) — a signed overflow occurred.

Most arithmetic instructions set these as a side effect. Two instructions exist purely to set
flags without storing a result:

- `cmp a, b` — computes `a - b`, sets flags, discards the result. (Used before conditional jumps.)
- `test a, b` — computes `a & b`, sets flags, discards the result. (`test rax, rax` is the
  idiomatic "is rax zero?" check.)

## Jumps

- `jmp target` — unconditional jump.
- Conditional jumps read the flags. Crucially, there are **separate signed and unsigned**
  families:

Signed comparisons (use SF/OF):
- `je`/`jz` (equal/zero), `jne`/`jnz` (not equal)
- `jl` (less), `jle` (less-or-equal), `jg` (greater), `jge` (greater-or-equal)

Unsigned comparisons (use CF):
- `jb` (below), `jbe` (below-or-equal), `ja` (above), `jae` (above-or-equal)

<Aside kind="insight">
The signed/unsigned split here is the machine-level root of the comparison pitfall from Module
1. The *same* `cmp` sets the flags; the compiler chooses `jl` (signed) or `jb` (unsigned) based
on the operand types. Pick the wrong type in C and you get the wrong jump.
</Aside>

## if / else

```c
int max(int a, int b) { return a > b ? a : b; }
```
A branchy compilation:
```
max:
    cmp edi, esi      ; compare a, b  (a in edi, b in esi)
    jg  .a_bigger     ; if a > b (signed), jump
    mov eax, esi      ; else result = b
    ret
.a_bigger:
    mov eax, edi      ; result = a
    ret
```

## Loops

A `for`/`while` becomes a compare-and-jump at the bottom (or top). A counting loop:

```c
int sum_n(int n) { int s = 0; for (int i = 0; i < n; i++) s += i; return s; }
```
```
sum_n:
    xor eax, eax        ; s = 0  (xor reg,reg is the idiom for zero)
    xor ecx, ecx        ; i = 0
.loop:
    cmp ecx, edi        ; compare i, n
    jge .done           ; if i >= n, exit
    add eax, ecx        ; s += i
    inc ecx             ; i++
    jmp .loop
.done:
    ret
```

## Conditional moves (cmov): branchless selection

Branches can be expensive when the CPU mispredicts them. For simple selections, compilers emit
**conditional moves** that have no branch at all:

```
; max(a,b) branchless:
    mov eax, esi      ; eax = b
    cmp edi, esi      ; compare a, b
    cmovg eax, edi    ; if a > b, eax = a
    ret
```

`cmovg` copies only if the condition holds — no jump, no misprediction risk. The trade-off:
`cmov` always evaluates both values, so it's a win only when both are cheap and the branch is
unpredictable.

## setcc: turning a condition into a 0/1

When you need a **boolean value** (not a jump), the `setcc` family writes 1 or 0 into a byte register
based on the flags — the machine form of C's `a < b` yielding an `int`:

```
; int is_neg(int x) { return x < 0; }
    test edi, edi
    sets al          ; al = SF (1 if negative)
    movzx eax, al    ; zero-extend the byte to a full int
    ret
```

`setl`, `sete`, `setb`, etc. mirror the jump conditions. This is another branchless building block.

## switch: jump tables

A dense `switch` doesn't become a chain of `cmp`/`je`. The compiler builds a **jump table** — an
array of code addresses — and does an indirect jump:

```
; switch(x) with cases 0..3
    cmp   edi, 3
    ja    .default        ; unsigned compare catches x<0 and x>3 in one check
    jmp   [table + rdi*8]  ; indirect jump to the case's address
table: .quad .case0, .case1, .case2, .case3
```

This turns an N-way branch into **O(1)** (one bounds check + one indirect jump) instead of O(N)
comparisons. Sparse switches may instead compile to a binary search of ranges. Note the single
**unsigned** `ja` bounds check cleverly rejects both negative and too-large indices at once.

## Recap

- Flags (ZF, SF, CF, OF) summarize the last operation; `cmp` and `test` set them deliberately.
- Conditional jumps come in **signed** (`jl/jg/...`) and **unsigned** (`jb/ja/...`) families —
  the machine root of the C comparison pitfall.
- `if`/loops are compare-and-jump; `xor reg,reg` zeros a register.
- `cmov` selects without branching, dodging mispredictions for simple cases.
