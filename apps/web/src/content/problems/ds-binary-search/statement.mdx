Implement `lower_bound` — binary search for the **first element ≥ a key**. It's the version of
binary search that actually shows up in practice (it's the B-tree node search and `std::lower_bound`),
and getting the boundary conditions right is the whole challenge.

## Your task

In `lowerb.c`, implement:

```c
int lower_bound(const int *a, int n, int key);
```

The array `a[0..n)` is sorted **ascending**. Return the smallest index `i` such that
`a[i] >= key`, or `n` if every element is smaller. Run in **O(log n)**.

## Examples

```
a = [1, 3, 5, 7]
lower_bound(a, 4, 5)  -> 2   (a[2] == 5)
lower_bound(a, 4, 4)  -> 2   (first value >= 4 is 5)
lower_bound(a, 4, 0)  -> 0   (all are >= 0)
lower_bound(a, 4, 100)-> 4   (none qualify)
[2,2,2,2], key 2      -> 0   (leftmost match)
```

## Hints (no spoilers)

- Half-open range `[lo, hi)` with `lo = 0, hi = n`. While `lo < hi`, take `mid = lo + (hi - lo) / 2`;
  if `a[mid] < key`, set `lo = mid + 1`, else `hi = mid`. Return `lo`.
- Use `lo + (hi - lo) / 2` to avoid overflow, and never read `a[mid]` when the range is empty.
- Duplicates: because you move `hi` down on `>=`, you land on the **leftmost** match.
